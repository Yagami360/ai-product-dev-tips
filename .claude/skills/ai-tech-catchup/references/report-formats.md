# レポート種別ごとの出力フォーマット

4 種類のレポート（最新 / 週次 / 月次 / トピック別）それぞれの構成を定義する。
ユーザーの依頼から種別を判定し、該当するセクションの見出し構成を**そのまま**使ってレポート本文を書く。

共通ルール:
- 各項目は**簡潔で読みやすい箇条書き**で書く。冗長な説明は避ける。
- 各情報には**必ず出典 URL** を添える。日付・具体的な数値データも入れる。
- 当年（実行時点の年）の最新リアルタイム情報を優先する。古い情報や推測で埋めない。

種別ごとのデフォルト件数・ラベル・Issue タイトル:

| 種別 | news_count | ラベル | Issue タイトル例 |
|------|-----------|--------|------------------|
| 最新 (latest) | 20 | `report` | `🤖 AI Tech Catchup Report - 2026-06-04` |
| 週次 (weekly) | 10 | `weekly-report` | `📊 AI Tech Catchup Weekly Report - 2026年06月第1週` |
| 月次 (monthly) | 20 | `monthly-report` | `📈 AI Tech Catchup Monthly Report - 2026年06月` |
| トピック別 (topic) | 20 | `topic-report` | `🎯 AI Tech Catchup Topic Report: AI Agent - 2026-06-04` |

ユーザーが件数を指定した場合はそれを優先する。

### 期間の計算（週次・月次のみ）

「前日」を終点に固定して計算する（実行日当日は含めない）。

- 週次: 前日から遡って 7 日間。例 `2026-05-28 ~ 2026-06-03`、見出し用は「2026年05月28日から2026年06月03日までの過去1週間」。週番号は `(実行日の日 - 1) // 7 + 1` で「YYYY年MM月第N週」。
- 月次: 前日から遡って 30 日間。例 `2026-05-05 ~ 2026-06-03`、見出し用は「2026年05月05日から2026年06月03日までの過去1ヶ月」。

見出し中の `{week_period}` / `{month_period}` は、**必ず上記の「YYYY年MM月DD日から…までの過去1週間/1ヶ月」という日付入りの表現**に置き換える（「過去1ヶ月」だけに省略しない）。週次と月次で表記を揃える。

## トピック別レポート (topic)

`{topic}` をユーザー指定のトピック名に置き換える。トピックの定義・背景から、課題・展望まで深く掘り下げる。

```markdown

### 🎯 トピック概要
[「{topic}」の定義、背景、重要性、現在の位置づけ]

### 🔥 重要ニュース（上位{news_count}件）
[「{topic}」に関する具体的なニュースタイトルと日付、詳細説明、URL付き]

### 📈 技術トレンド
[「{topic}」の最新技術トレンドと実例、数値データ付き]

### 🚀 技術的ハイライト
[「{topic}」の新技術、画期的な進歩、ブレークスルー]

### 🔄 技術的変化
[「{topic}」における技術的進化・革新・主要な変更点]

### 📈 市場動向
[「{topic}」の市場規模・成長率・競合状況・ビジネス機会]

### 🔧 オープンソース（OSS）動向
[「{topic}」関連のGitHub・Hugging Faceプロジェクト、主要なOSSツール]

### 📱 SNS・コミュニティ動向
[「{topic}」に関するSNSの議論、専門家の意見、コミュニティの反応]

### 💡 開発者向けポイント
[「{topic}」を使った実装例、ベストプラクティス、リソース、チュートリアル]

### 🔬 研究者向けポイント
[「{topic}」の研究動向、主要論文、データセット、ベンチマーク、実験手法]

### 🔗 主要リソース
[「{topic}」関連の公式ドキュメント、ライブラリ、フレームワーク、ツール]

### ⚠️ 課題と制限
[「{topic}」の現在の課題、技術的制約、未解決の問題]

### 🔮 今後の展望
```

## Issue 本文テンプレート

レポート本文を、以下のヘッダ・フッタで囲んで Issue 本文にする。`<絵文字><見出し>` は種別に合わせる。

```markdown
# <絵文字> AI Tech Catchup <種別名> Report

- レポート日時: `YYYY-MM-DD HH:MM`
- 調査期間: `YYYY-MM-DD ~ YYYY-MM-DD`   ← 週次・月次のみ。最新・トピックでは省略
- 使用モデル: `<このセッションのモデル名>`
- thinking / effort level: `<このセッションの thinking / effort level>`
- 調査方法: `<この調査で実際に使ったツールを列挙。例: WebSearch, WebFetch, GitHub MCP, Hugging Face MCP, arXiv MCP, /deep-research>`
- トピック: `<topic>`                    ← トピック別のみ

---

<レポート本文>

---

*このレポートは AI Tech Catchup スキルによって自動生成されました。*
```
