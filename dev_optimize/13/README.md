## コマンドの全体像（系統別）

Claude Code の操作系は、大きく次の 4 系統に分けられる。②スラッシュコマンドはさらに目的別に分類できる。

| 系統 | 実行タイミング | 例 |
| :-- | :-- | :-- |
| ① CLI 起動フラグ | ターミナルで `claude` を起動する時に指定 | `-p` / `-c` / `-r` / `--model` / `--permission-mode` |
| ② スラッシュコマンド | セッション中に `/` で実行 | `/plan` / `/compact` / `/code-review` / `/deep-research` |
| ③ 入力プレフィックス | 入力の先頭文字で挙動を変える | `/`（コマンド）/ `!`（シェル）/ `@`（ファイル参照） |
| ④ キーボードショートカット | 対話中のキー操作 | `Shift+Tab` / `Esc` / `Ctrl+R` / `Ctrl+O` |

## スラッシュコマンド一覧表（系統別）

セッション中に `/` に続けて入力するスラッシュコマンドを、開発ワークフローに沿った系統別に整理する（各表はカテゴリ順にソート）。`<arg>` は必須引数、`[arg]` は任意引数を表す。**Skill** はバンドルされた Skill、**Workflow** はバンドルされた動的ワークフローを示す。

「活用済み」列は、Yagami360 が実際によく使っているコマンドを ✅（活用済み）、それ以外を ❌（未活用）で示す（好みや用途によって変わるので参考程度に）。

## CLI 起動フラグ（よく使うもの）

ターミナルから `claude` を起動する時に指定するフラグ。ヘッドレス実行や、モデル・権限モードの指定、セッションの継続などに使う。

| カテゴリ | フラグ / コマンド | 内容 | 補足 |
| :-- | :-- | :-- | :-- |
| 起動 | `claude` | 対話セッションを開始 | `claude "質問"` で初期プロンプト付き |
| ヘッドレス | `claude -p "query"` / `--print` | 応答を出力して終了（SDK/ヘッドレスモード） | `cat file \| claude -p "..."` でパイプ入力 |
| 継続 | `claude -c` / `--continue` | 直近の会話を継続 | `-c -p` でヘッドレス継続 |
| 再開 | `claude -r "<session>"` / `--resume` | セッションを ID/名前で再開 | |
| モデル | `--model <model>` | モデル指定（`sonnet`/`opus`/`haiku`/`fable`/フル ID） | |
| 推論 | `--effort <level>` | effort レベル（`low`〜`max`） | |
| 役割 | `--agent <name>` | カスタムサブエージェントとしてセッションを起動 | システムプロンプトを置換 |
| 権限 | `--permission-mode <mode>` | 権限モード指定（`default`/`acceptEdits`/`plan`/`auto`/`dontAsk`/`bypassPermissions`） | |
| 権限 | `--dangerously-skip-permissions` | 権限プロンプトをスキップ | `= --permission-mode bypassPermissions` |
| ディレクトリ | `--add-dir <paths>` | 追加の作業ディレクトリを許可 | |
| 分離 | `--worktree <name>` / `-w` | 独立した git worktree で起動 | |
| セッション | `--fork-session` | 新しいセッション ID で開始 | `--resume`/`--continue` と併用 |
| 出力 | `--output-format <fmt>` | 出力形式（`text`/`json`/`stream-json`） | print モード用 |
| 保守 | `claude update` | 最新版に更新 | |
| MCP | `claude mcp` | MCP サーバーを設定 | |

## キーボードショートカット・入力プレフィックス

対話中に使うキーボードショートカットと、入力の先頭に付けるプレフィックス。プラットフォーム・ターミナルによって挙動が異なる場合がある（特に macOS の `Option`/`Alt` 系は [Option を Meta として設定](https://code.claude.com/docs/en/terminal-config)が必要）。

### 入力プレフィックス（入力の先頭文字）

| プレフィックス | 内容 | 補足 |
| :-- | :-- | :-- |
| `/` | コマンド／Skill を実行 | `/` で候補メニュー表示 |
| `!` | シェルモード（コマンドを直接実行し出力を文脈に追加） | `! npm test` `! git status` など。出力に対し Claude が自動で応答（v2.1.186〜） |
| `@` | ファイルパスの参照（オートコンプリート） | プロンプト中でファイルを指定 |

### 主なキーボードショートカット

| カテゴリ | ショートカット | 内容 |
| :-- | :-- | :-- |
| モード | `Shift+Tab` | 権限モードを循環切替（`default`→`acceptEdits`→`plan`→有効化した `auto`/`bypassPermissions` など） |
| 中断 | `Esc` | Claude の応答・ツール実行を中断（ダイアログ表示中はダイアログを閉じる） |
| 巻き戻し | `Esc` `Esc`（2回） | 入力があれば下書きクリア、空なら [rewind メニュー](https://code.claude.com/docs/en/checkpointing)を開く |
| 中断/終了 | `Ctrl+C` | 実行中なら中断、何もなければ入力クリア→2回目で終了 |
| 終了 | `Ctrl+D` | セッション終了（EOF） |
| 履歴 | `Ctrl+R` | コマンド履歴の逆検索（`Ctrl+R` で古いマッチへ、`Ctrl+S` で範囲切替） |
| 履歴 | `↑` / `↓`（`Ctrl+P`/`Ctrl+N`） | カーソル移動／コマンド履歴の移動 |
| 表示 | `Ctrl+O` | トランスクリプトビューア（詳細なツール利用）の切替 |
| 表示 | `Ctrl+T` | Claude の ToDo チェックリスト表示の切替 |
| 表示 | `Ctrl+L` | 画面の再描画（表示崩れの復旧） |
| バックグラウンド | `Ctrl+B` | 実行中の Bash コマンド・エージェントをバックグラウンド化（tmux は2回） |
| 画像 | `Ctrl+V`（iTerm2 は `Cmd+V`） | クリップボードの画像を貼り付け（`[Image #N]`） |
| エディタ | `Ctrl+G`（または `Ctrl+X Ctrl+E`） | 既定のテキストエディタでプロンプトを編集 |
| モデル | `Option+P` / `Alt+P` | プロンプトを保ったままモデル切替 |
| 思考 | `Option+T` / `Alt+T` | 拡張思考（extended thinking）の切替 |
| fast | `Option+O` / `Alt+O` | fast モードの切替 |
| 改行 | `Shift+Enter` / `Ctrl+J` / `\`+`Enter` | 複数行入力（`Shift+Enter` は一部ターミナルで要 `/terminal-setup`） |

> Vim 風のキー操作は、`/config` → Editor mode で有効化できる（旧 `/vim` コマンドは v2.1.92 で削除）。
